nginx+uwsgi部署

浏览器发起请求
nginx接收请求 并对请求解包 分析
如果访问的是静态文件 就根据nginx的配置文件目录，返回请求的资源
如果是动态请求 nginx就通过配置文件 将请求传递给uwsgi uwsgi将接收到的包进行处理
wsgi 根据请求调用django工程的某个文件或者函数 处理完后django将结构返回给wsgi
wsgi将返回值进行打包 转发给uWSGI
uWSGI接收后转发给nginx nginx最后将返回值给客户端

mysql主从服务器

1.安装mysql
sudo docker load -i ~/Downloads/下的mysql压缩包
2.在/home/python中创建目录
mkdir mysql_slave
cd mysql_slave
mkdir data
cp -r /etc/mysql/mysql.conf.d ./
3.修改配置文件
sudo vi ~/mysql_slave/mysql.conf.d/mysqld.cnf
port = 8306
general_log = 0
server_id = 2
4.创建docker
sudo docker run –name mysql-slave -e MYSQL_ROOT_PASSWORD=mysql -d –network=host -v /home/python/mysql_slave/data:/var/lib/mysql -v /home/python/mysql_slave/mysql.conf.d:/etc/mysql/mysql.conf.d mysql:5.7.22
5.测试
mysql -uroot -pmysql -h 192.168.247.129 –port=8306
6.备份
mysqldump -uroot -pmysql -all-databases –lock-all-tables > ~/master_db.sql
7.将数据导入到docker容器中
mysql -uroot -pmysql -h 192.168.247.129 –port=8306 < ~/master_db.sql
再运行5测试 看下数据库中的内容是不是多了

配置主服务器

1.修改配置文件
sudo vim /etc/mysql/mysql.conf.d/mysqld.cnf
2.重启mysql服务
sudo service mysql restart
3.进入主服务器的mysql
创建用于从服务器同步数据的使用的账号
mysql -uroot -pmysql
grant replication slave on . to ‘slave’@’%’ identified by ‘slave’;
flush privileges;
获取主服务器的二进制日志信息
show master status;
4.进入docker中的mysql
myslq -uroot -pmysql -h192.168.247.129 –port=8306
执行
change master to master_host=’127.0.0.1’, master_user=’slave’, master_password=’slave’,master_log_file=’mysql-bin.000006’, master_log_pos=590;
master_log_file需要修改
5.启动slave服务器 并查看同步状态
start slave
show slave status \G:
看到Slave_IO_Running：Yes
Slave_SQL_Running: Yes
表示同步已正常运行
